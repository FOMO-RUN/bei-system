# 悖 (BEI) — 语言裁决账本

> 记录每个关键命名点的证据、结论和策略

## 使用说明

每当遇到命名决策时：
1. 先查本账本是否有先例
2. 无先例则做 Unicode 预检测
3. 记录裁决到本账本

---

## 裁决记录

### #001 Rust Crate 名称

| 属性 | 值 |
|------|-----|
| 命名点 | Cargo.toml 中的 `[package] name` |
| 检测日期 | 2026-01-29 |
| 结论 | **必须 ASCII** |
| 证据 | Cargo 官方文档：crate 名仅允许 `[a-zA-Z0-9_-]` |
| 策略 | 使用 ASCII 名 + `docs/词汇映射表.yaml` 中文映射 |
| 影响范围 | 所有 Rust crate |

---

### #002 Git 分支名

| 属性 | 值 |
|------|-----|
| 命名点 | `git branch` / `jj bookmark` |
| 检测日期 | 2026-01-29 |
| 结论 | **可以中文** |
| 证据 | Git 2.x 默认 UTF-8，Windows 下测试通过 |
| 策略 | 直接使用中文分支名，如 `功能/回放系统` |
| 影响范围 | 版本控制分支命名 |

### #003 目录与文件名

| 属性 | 值 |
|------|-----|
| 命名点 | 文件系统路径 |
| 检测日期 | 2026-01-29 |
| 结论 | **可以中文** |
| 证据 | Windows NTFS + Rust `std::fs` 支持 Unicode |
| 策略 | 文档/配置类文件用中文名，代码文件按语言惯例 |
| 影响范围 | 项目所有文件路径 |
| 例外 | `.gitignore`, `Cargo.toml` 等工具强制要求的文件名 |
